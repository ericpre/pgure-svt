PGURE-SVT

Singular value thresholding for denoising video microscopy data

PGURE-SVT is an algorithm designed to denoise image sequences acquired in microscopy. It exploits the correlations between consecutive frames to form low-rank matrices, which are then recovered using a technique known as nuclear norm minimization. An unbiased risk estimator for mixed Poisson-Gaussian noise is used to automate the selection of the regularization parameter, while robust noise and motion estimation maintain broad applicability to many different types of microscopy. The algorithm is described in detail in:

Furnival T, Leary R, Midgley PA. (2016). Denoising time-resolved microscopy sequences with singular value thresholding. Manuscript in preparation.

Installation

Dependencies

PGURE-SVT makes use of several 3rd-party libraries, which need to be installed first.

  • CMake 2.8+
  • Armadillo 6.400+
  • NLopt 2.4.2+
  • LibTIFF - for the standalone executable only

Compiling from source

To build PGURE-SVT, unpack the source and use the following commands 

$ tar -xzf pgure-svt.tar.gz
$ cd pgure-svt
$ mkdir build
$ cd build
$ cmake ..
$ make

This will generate the appropriate shared library for the Python wrapper in the build directory. The standalone executable is not built by default, as it requires LibTIFF to be installed. To compile the executable as well as the library, use:

$ cmake -DBUILD_EXECUTABLE=ON ..
$ make

Using PGURE-SVT with Python

An example file, demo.py, is provided to show users how the algorithm can be linked with the HyperSpy multi-dimensional data analysis toolbox, which provides a number of useful features including data visualization and data import from a number of microscopy file formats.

import pguresvt

# Initialize the algorithm
svt = pguresvt.SVT(patchsize=4,
                   length=15,
                   optimize=True,
                   tol=1e-6)

# Run the algorithm on the data X
# and get the denoised output Y
Y = svt.denoise(X)

Using PGURE-SVT from the command line

The PGURE-SVT standalone executable uses a simple command-line interface along with a separate parameter file. It is convenient to use on remote computing clusters. The parameter file allows the user to customize various options of the PGURE-SVT algorithm. An example file, param.svt is provided. 

$ ./PGURE-SVT param.svt

Note that the standalone executable currently only supports TIFF sequences.
